# 🛡 AI-Based Military Intelligence Dashboard

A Streamlit dashboard for exploring, visualizing, and forecasting terrorism
incident data from the **Global Terrorism Database (GTD)**, with two
supervised ML models (attack-type and threat-level prediction) and several
unsupervised analysis tools (hotspot clustering, anomaly detection,
group/country network analysis).

> Built for open-source research / coursework use on publicly available GTD
> data. Not connected to any real intelligence system.

## 1. Setup

```bash
cd AI_Military_Intelligence
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

The GTD CSV is already included at `data/globalterrorism.csv`. If you
replace it with a different export, keep that filename and encoding
(`latin1`), or update `DATA_PATH` in `utils/data_loader.py`.

## 2. Train the ML models (once)

```bash
python train_models.py
```

This trains and saves two models into `models/`:

- **Attack Type Prediction** — Random Forest predicting `attacktype1_txt`
  from country, region, weapon, target, group, success/suicide flags, and
  casualty counts.
- **Threat Level Prediction** — Random Forest predicting LOW/MEDIUM/HIGH
  severity from *pre-attack* context only (country, region, attack type,
  weapon, target, suicide flag) — casualty counts are **not** used as
  inputs since they define the label (see "Fixes" below).

## 3. Run the app

```bash
streamlit run Home.py
```

Demo login: **analyst** / **gtd2024**

## What changed vs. the original scripts

**Bug fixes**
- `utils/data_loader.py` didn't exist — several pages imported it but it
  was never provided; now implemented with caching and shared cleaning.
- `train_attack_model.py` pointed at `data/globalterrorism.csv` but the
  supplied file was named differently — fixed and consolidated into
  `train_models.py`.
- `5_🚨_Threat_Level.py` retrained a Random Forest on the full ~180k-row
  dataset **on every page load** — now loads a model trained once ahead
  of time via `st.cache_resource`.
- **Data leakage** in the original threat-level model: it used
  `nkill`/`nwound` as *both* the input features and the basis for the
  label, producing a meaningless ~100% "accuracy". The model now predicts
  severity from pre-attack context only (~70% accuracy — a real result).
- `4_🤖_Attack_Prediction.py` reloaded the dataset and re-ran predictions
  on every rerun regardless of form submission; prediction logic is now
  gated behind the submit button.
- Duplicate/inconsistent entry points (`app.py` vs `1___Home.py`) merged
  into a single `Home.py`.
- The Settings page's controls didn't persist or affect anything; they
  now write to `st.session_state` and are actually applied (theme, global
  filters, etc.).
- Unbounded marker maps (100k+ points) could freeze the browser — the
  map pages now sample or offer a density heatmap for large selections.

**New features (not in the original project)**
- 🔐 Simple login gate (`utils/auth.py`).
- 🧭 **Hotspot Clustering** page — DBSCAN spatial clustering to rank
  geographic hotspots by incident density and severity.
- 🕸 **Network Analysis** page — group↔country bipartite graph to surface
  transnational organizations operating across multiple countries.
- 📈 Automatic **anomaly detection** (Isolation Forest) on the yearly
  forecast trend, flagging unusually high/low years.
- Global sidebar filters (year range, countries) shared across pages via
  `st.session_state`.
- Feature-importance chart on the Attack Prediction page.
- Dark theme actually applied via CSS injection tied to the Settings page.

## Disclaimer

This project analyzes publicly available historical incident data for
research/educational purposes. It does not provide operational guidance
for planning, conducting, or evading detection of any attack.
